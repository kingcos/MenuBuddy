## 概览

v2 新增两个相互关联的系统，鼓励用户持续使用 MenuBuddy：

```
┌─────────────────────┐     解锁槽位         ┌─────────────────────┐
│      成长系统        │ ──────────────────▶ │      装扮系统        │
│                     │                     │                     │
│  XP → 等级 → 点数  │     升级奖励         │  物品 → 装备 →      │
│                     │ ──────────────────▶ │  精灵修饰器          │
│  5 种经验来源        │                     │                     │
│  属性加成           │     更换角色         │  5 槽位，40+ 物品    │
│  每日登录           │ ◀─── Lv.5+ ──────▶ │  自定义创作          │
└─────────────────────┘                     └─────────────────────┘
```

**设计原则：**
- 非侵入式——系统丰富体验但绝不阻断现有功能
- 渐进式展示——功能随等级逐步解锁
- 可插拔——装扮物品数据驱动，易于扩展
- 可逆——完全重置可恢复初始状态
- **可选启用**——设置中的总开关，**默认关闭**。不喜欢该功能的用户完全不受影响

## 成长系统

**源文件：** `Sources/MenuBuddy/Models/ProgressionSystem.swift`

### 经验来源

用户通过 5 种互动方式获取经验值：

| 来源 | XP | 触发条件 |
|------|-----|---------|
| 抚摸 | 5 | 每次点击伙伴精灵 |
| 每日登录 | 20 | 每天首次启动应用（自动领取） |
| 系统事件 | 3 | 系统监控触发（CPU/内存/网络/电池） |
| 应用切换 | 1 | 切换到可识别的应用（IDE、终端、浏览器等） |
| AI 反应 | 8 | AI 生成一条回复 |
| 里程碑 | 15 | 达到抚摸次数里程碑（1, 5, 10, 25, 50, 100） |

### 等级公式

```
等级 = floor(sqrt(总XP / 10))
```

这产生一条平滑递减的增长曲线：

| 等级 | 所需总 XP | 升级增量 |
|------|-----------|---------|
| 0 | 0 | — |
| 1 | 10 | 10 |
| 2 | 40 | 30 |
| 3 | 90 | 50 |
| 4 | 160 | 70 |
| 5 | 250 | 90 |
| 6 | 360 | 110 |
| 7 | 490 | 130 |
| 8 | 640 | 150 |
| 9 | 810 | 170 |
| 10 | 1000 | 190 |

**当前等级内的进度**计算方式：

```
进度 = (当前XP - 当前等级所需XP) / (下一等级所需XP - 当前等级所需XP)
```

该值驱动面板头部的 XP 进度条。

### 属性加点

每升一级获得 **+2 属性点**，可分配到 5 项属性中的任意一项：

- 调试力（DEBUGGING）
- 耐心值（PATIENCE）
- 混乱度（CHAOS）
- 智慧值（WISDOM）
- 嘴贱度（SNARK）

**有效属性值** = 基础属性（由机器绑定的随机值决定） + 分配的加成，上限 100。

通过面板中每个属性条旁边的 `+` 按钮分配点数。未分配的点数在 XP 条区域显示为 `+N pts ↓` 标识。

### 更换角色

**等级 5 以上**时，用户可通过物种图鉴更换伙伴的物种：

1. 打开物种图鉴（工具栏网格图标）
2. 点击任意物种进行预览
3. 非当前物种会显示「更换角色」按钮
4. 确认对话框 → 物种立即更换

更换结果以 `speciesOverride` 持久化在 UserDefaults 中。原始的机器绑定随机值被保留，重置时恢复。

### 槽位解锁

装扮槽位在特定等级解锁：

| 等级 | 解锁槽位 |
|------|---------|
| 0 | 帽子 |
| 2 | 眼睛 |
| 3 | 配饰 |
| 5 | 光环 |
| 8 | 边框 |

未解锁的槽位在装扮界面中显示为灰色，带锁图标和「Lv.N」标签。

### 反作弊

| 机制 | 数值 | 说明 |
|------|------|------|
| **每日 XP 上限** | 200 XP/天 | 每日登录奖励（20 XP）不计入 |
| **抚摸冷却** | 2 秒 | 防止连续点击刷经验 |
| **系统事件冷却** | 10 秒 | 与系统监控轮询频率匹配 |
| **应用切换冷却** | 30 秒 | 防止快速 Alt-Tab 刷经验 |
| **LLM 冷却** | 15 秒 | 防止 API 滥用 |

- 冷却期内的 XP 请求被**静默拒绝**（不发放 XP，不记录事件）
- 超出每日上限的 XP 被**截断**至剩余额度
- 每日统计在午夜自动重置（按日历日）
- 冷却时间仅存于内存（应用重启后重置——有意为之的宽容设计）

### 全局开关

整个成长 + 装扮系统受总开关控制：

- **设置 → 成长 → 「启用成长与装扮系统」**
- **默认：关闭**——新用户享受经典体验
- 关闭时：不发放 XP、隐藏等级/XP条/装扮按钮、属性只显示基础值、精灵不应用装扮修饰
- 开启时：完整 v2 体验
- 持久化在 UserDefaults（`companion.progressionEnabled`）

## 装扮系统

**源文件：** `Sources/MenuBuddy/Models/CosmeticSystem.swift`

### 架构设计

```
┌──────────────────┐     ┌──────────────────┐     ┌──────────────────┐
│  CosmeticCatalog │     │ CosmeticInventory│     │  SpriteRenderer  │
│  （静态物品）     │     │  （持有 + 装备）  │     │  （应用修饰）     │
│  40+ 物品        │────▶│  ownedItemIds    │────▶│                  │
│  5 种稀有度      │     │  equipped 映射   │     │  renderSprite()  │
│  5 种槽位        │     │  customItems[]   │     │  renderFace()    │
└──────────────────┘     └──────────────────┘     └──────────────────┘
                                │
                                ▼
                         ┌──────────────────┐
                         │ SpriteModifier   │
                         │ （缓存，仅在装备/ │
                         │  卸载时重建）     │
                         └──────────────────┘
```

核心设计：`CosmeticItem` 包含一个 `SpriteModifier` 结构体，声明式地描述如何修改 ASCII 精灵。渲染器消费所有已装备物品的合并修饰器。

### 槽位与物品

每个物品归属于唯一一个槽位：

| 槽位 | 修饰字段 | 视觉效果 |
|------|----------|---------|
| **帽子** | `hatLine`（12 字符字符串） | 替换精灵的空白首行 |
| **眼睛** | `eyeChar`（单个字符） | 替换精灵帧中的 `{E}` 占位符 |
| **配饰** | `accessoryLeft` / `accessoryRight` | 添加到精灵中间行的左侧/右侧 |
| **光环** | `auraTop` / `auraBottom` | 在精灵上方/下方添加新行 |
| **边框** | `frameLeft` / `frameRight` | 包裹精灵的每一行 |

### 物品目录

**42 件物品**，横跨 5 种稀有度：

| 稀有度 | 掉落权重 | 物品数 | 颜色 |
|--------|---------|--------|------|
| 普通 | 50% | 5 | 灰色 `#888` |
| 优秀 | 30% | 10 | 绿色 `#22c55e` |
| 稀有 | 13% | 11 | 蓝色 `#3b82f6` |
| 史诗 | 5% | 9 | 紫色 `#a855f7` |
| 传说 | 2% | 7 | 金色 `#f59e0b` |

**帽子（13 件）：** 无、皇冠、高礼帽、螺旋桨帽、光圈、巫师帽、毛线帽、小鸭子、海盗帽、天线、小花、厨师帽、派对帽、忍者头带

**眼睛（8 件）：** 默认、星星 ✦、爱心 ♥、钻石 ◆、螺旋 @、闪电 ⚡、月亮 ☽、叉叉 ×、圆环 ◉

**配饰（8 件）：** 蝴蝶结、宝剑、盾牌、魔法棒、围巾、小旗、气球、吉他

**光环（6 件）：** 闪闪发光、火焰、爱心、飘雪、音乐、彩虹

**边框（5 件）：** 方括号、竖线、星星框、箭头框、钻石框

### 自定义帽子创作器

用户可以创建自己的帽子物品：

1. 打开装扮面板 → 选择帽子槽位 → 点击「创建」
2. 输入名称和 12 字符的 ASCII 图案
3. 实时预览帽子在伙伴头上的效果
4. 创建的物品存储在 `CosmeticInventory.customItems` 中
5. 自定义物品显示画笔图标，可通过右键菜单删除

自定义物品仅在本地持久化，不可导出给其他用户（目录验证机制阻止导入任意数据）。

### 随机掉落

物品通过两种方式获取：

**抚摸掉落（每次抚摸 5% 概率）：**
1. 从符合条件的物品中随机抽取（解锁等级 ≤ 用户等级，且未拥有）
2. 按稀有度权重加权
3. 如果没有符合条件的物品则不掉落

**升级奖励（必得）：**
1. 同样的加权随机，从符合条件的物品池中抽取
2. 每次升级一件

### 导入与导出

物品可在用户间分享：

- **导出：** 将 `CosmeticItem` 序列化为 base64 编码的 JSON → 复制到剪贴板
- **导入：** 解码 base64，验证物品 ID 存在于目录中，添加到背包
- 安全性：仅允许导入目录中已有的物品（防止注入任意数据）

### 精灵渲染

`renderSprite()` 函数按以下顺序应用修饰：

1. **眼睛覆盖：** 装扮眼睛 > 眨眼 "-" > 默认眼睛字符
2. **帽子行：** 装扮帽子 > 伙伴自带帽子 > 无
3. **移除空白帽子槽**（如果物种所有帧的首行都是空白）
4. **配饰：** 在中间行添加左/右装饰字符串
5. **光环：** 在精灵上方/下方添加装饰行
6. **边框：** 在所有行外侧包裹左/右字符

**菜单栏面部**（`renderFace()`）同样支持装扮眼睛：
- 优先级：系统触发眼睛（压力）> 装扮眼睛 > 眨眼 > 默认
- 装扮眼睛会抑制眨眼闪烁

**性能：** 合并后的 `SpriteModifier` 被缓存，仅在装备/卸载时重建，而非每帧渲染时。

### 行为修饰器（增强版装扮）

除了视觉变化，史诗和传说物品还携带**行为修饰器**，影响伙伴的行为表现：

| 字段 | 类型 | 效果 |
|------|------|------|
| `exclusiveQuips` | `[String]?` | 物品专属台词，混入台词轮播 |
| `equipAnnouncement` | `String?` | 装备时显示的一次性语音 |
| `idleSequenceOverride` | `[Int]?` | 替换默认待机动画序列 |

**携带行为修饰器的物品：**

| 物品 | 稀有度 | 专属台词 | 装备语音 | 动画覆盖 |
|------|--------|---------|---------|---------|
| 巫师帽 | 史诗 | 3 条魔法台词 | "I feel powerful" | — |
| 小鸭子 | 传说 | 4 条鸭子台词 | "we are one" | 更快更活泼 |
| 海盗帽 | 史诗 | 3 条海盗台词 | "pirate mode activated" | — |
| 厨师帽 | 史诗 | 3 条烹饪台词 | "now we're cooking" | — |
| 忍者头带 | 传说 | 4 条潜行台词 | "I am shadow" | 更慢更隐蔽 |

行为修饰器接入 `AnimationEngine`：
- 专属台词追加到洗牌台词池
- 待机序列覆盖替换 `idleSequence` 用于帧/眨眼计算
- 装备语音在装扮面板预览区显示为对话气泡

## 集成点

两套系统与 MenuBuddy 现有架构的集成点：

| 组件 | 集成方式 |
|------|---------|
| **CompanionStore** | 持有 `ProgressionSystem` 和 `CosmeticSystem` 实例；发布 `level`、`levelProgress`、`totalXP`、`availablePoints`；将 XP 发放接入现有操作 |
| **PopoverView** | 头部显示等级徽章 + XP 进度条；升级庆祝弹窗；装扮掉落通知 |
| **CompanionView (StatsView)** | 显示有效属性值（基础 + 加成）；有可用点数时显示 `+` 分配按钮 |
| **CompanionView (AnimationEngine)** | `onAppContextSwitch` 闭包发放应用切换 XP |
| **SpriteRenderer** | `renderSprite()` 新增 `cosmeticModifier` 参数；`renderFace()` 支持装扮眼睛覆盖 |
| **AppDelegate** | 右键菜单显示等级；工具提示包含等级；关于对话框显示等级 + XP；菜单栏面部反映装扮眼睛 |
| **SettingsView** | 新增「成长」区块，显示等级、XP 进度条、槽位解锁状态 |
| **SpeciesAtlasView** | Lv.5+ 显示「更换角色」按钮，带确认对话框 |
| **HelpView** | 新增成长和装扮两条使用提示 |
| **LLMService** | 系统提示词包含等级和 XP，使 AI 反应更具个性 |
| **重置流程** | 清除成长进度、装扮背包、物种覆盖和新手引导标记 |

## 数据持久化

所有数据存储在 **UserDefaults** 中：

| 键名 | 类型 | 内容 |
|------|------|------|
| `progression.state` | JSON Data | `ProgressionState`（XP、加成、事件、每日领取） |
| `cosmetic.inventory` | JSON Data | `CosmeticInventory`（持有 ID、装备映射、自定义物品） |
| `companion.speciesOverride` | String | 物种 rawValue（nil = 使用机器随机值） |
| `onboarding.xpSeen` | Bool | 首次 XP 引导消息已显示 |
| `onboarding.cosmeticsSeen` | Bool | 首次装扮面板引导已显示 |

所有键在伙伴重置时通过 `CompanionStore.resetCompanion()` 清除。
